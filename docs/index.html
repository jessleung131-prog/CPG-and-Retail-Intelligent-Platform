Forecasting Models

Dashboard API

FastAPI with 8 live endpoints, dynamic filtering, lru_cache data loading, and automatic reliability warnings when data is sparse.

Live Endpoints

GET /forecast/sales Prophet or XGBoost 90-day forecast model, channel, min 60 rows

GET /attribution/spend-efficiency ROAS, CPA, ROI per channel date range
GET /crm/funnel Stage counts, conversion rates, avg lag date range
GET /crm/by-source Pipeline breakdown by lead source date range
POST /cache/clear Invalidate lru_cache after data refresh —

Dynamic Filtering

Every endpoint accepts channel, category, start_date, and end_date query params. The apply_filters() helper validates and slices consistently across all routes.

Reliability Warnings

When a filtered slice returns fewer than 30 rows, the API automatically attaches a reliability_warning field to the response — surfacing data sparsity to the consumer.

Performance

Parquet files are loaded once via @lru_cache on startup. Subsequent requests hit the in-memory cache. The /cache/clear endpoint invalidates after any data refresh.

What Monitoring Covers

  • Data Freshness Flags when the latest record in any table exceeds the warning (3d) or critical (7d) threshold. First defence against silent ingestion failures.
  • Row Count Health Compares each new load to a rolling historical average. A 40%+ drop is a warning; 60%+ is critical. Catches partial loads and API quota issues.
  • Statistical Anomaly Detection Z-score outliers (|z| > 3), IQR outliers (1.5× fence), level-shift detection via rolling mean deviation, and null spike checks on every numeric column.
  • Slack Alerts All critical and warning alerts are dispatched to a configured Slack webhook with severity label, table name, and root-cause hint — no dashboard required.
